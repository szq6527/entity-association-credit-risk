# 项目成果材料

## 一、数据与问题定义

本项目面向上市公司信用风险预警，核心任务是预测企业在下一年是否发生评级下调。我们以 2010-2024 年 A 股上市公司为研究对象，构建了覆盖 3915 家公司的多源异构数据底座。

数据侧主要包含六类信息：公司基本信息与行业属性、财务三表、对外担保事件、股权与控制人关系、日频股价收益、信用评级事件。围绕“风险传导”这一业务逻辑，我们将原始表统一到公司-时间主键，完成证券代码标准化、日期对齐、异常值处理和关系映射，形成可直接建模的数据资产。

在关系建模上，当前形成 6 类年度关系：
- `guarantee`（上市公司之间担保）
- `shared_nonlisted_guarantee`（共同担保非上市主体）
- `equity_assoc`（股权关联）
- `equity_change`（股权变更事件关联）
- `co_controller`（同实控人关联）
- `market_corr`（基于股票收益相关性的市场关系）

该数据体系的价值在于：既保留财务与评级等“节点风险信号”，又显式刻画企业间“关系传播路径”，为后续图模型提供可解释、可扩展的输入基础。

## 二、代码工程与可复现成果

我们已经搭建端到端可复现工程流程，覆盖“清洗-建图-训练-评估”全链路。核心脚本链路为：
`prepare_data_stage1.py -> build_training_panel.py -> build_rating_task_dataset.py -> build_final_hetero_temporal_graph.py -> train_htgnn_dgl_improved.py`

工程实现上有三个特点：
- 标准化流程：通过 `scripts/run_all.sh` 一键串联主要步骤，降低重复实验成本；
- 配置化训练：关系组合、损失函数、窗口长度、模型维度均可参数化；
- 结果可追溯：每次实验自动输出 `config.json`、`metrics.json`、训练曲线与预测结果，支持对比与复盘。

目前项目已完成多关系时序图构建和改进版 HTGNN 训练验证，形成稳定产物目录：`processed/stage1/`、`processed/final_hetero_temporal_graph/`、`experiments_htgnn_dgl_improved/`。在此基础上，我们具备继续开展关系消融、参数优化和跨期稳定性验证的工程能力，能够支撑后续论文实验与业务化落地评估。
