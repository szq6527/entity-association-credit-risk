# 存在问题与下一步研究方案

当前项目已形成“多关系时序图 + 评级下调预测”的完整流程，但仍存在几类关键瓶颈。第一，关系信号强弱不均衡。`co_controller` 与 `market_corr` 边规模明显大于担保与股权类关系，模型训练时容易被高密度关系主导，削弱了高业务解释性的“担保传导”信号。第二，标签覆盖有限。监督目标依赖评级迁移，导致可训练样本集中在部分公司与年份，模型泛化到全市场时存在偏差风险。第三，部分关系噪声仍然较高，例如名称映射得到的关系边可能存在错配，且市场相关性关系受交易状态、极端收益、年度窗口长度等参数影响较大。第四，实验评估虽已覆盖 AUC、AP 与 P@K，但对稳定性（跨种子、跨窗口、跨年份漂移）验证仍不足，尚不能充分支撑结论鲁棒性。

下一步研究建议分三条线推进。其一，做“关系质量与贡献”系统评估：通过逐关系消融、关系分组加权、边采样与阈值敏感性分析，量化每类关系对 AP 与 P@K 的边际贡献，并据此重设关系权重。其二，强化时序建模与标签工程：引入多期目标（t+1 到 t+2）、滚动训练窗口与样本重加权，缓解单期标签稀疏与年份分布不稳问题。其三，完善数据治理：建立名称映射置信度规则、异常边审计清单与关系版本管理，确保每次构图可追溯、可复现。落地节奏上，建议先完成“6 关系消融 + 3 随机种子复现实验”，再进入结构优化（如关系门控或关系正则），最终以测试集 AP、Recall@5% 和跨期稳定性作为主决策指标。
